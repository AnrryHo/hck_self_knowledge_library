# 目标检测

# 摘要

本主题页基于为知笔记“AI研习图书馆 目标检测从入门到精通”下已入库的 8 篇笔记，覆盖目标检测项目结构、基础任务、滑动窗口、R-CNN、评估指标、SPPNet 和 Fast R-CNN。当前资料多为课程截图和简短文字，细节结论需要回查对应 raw Markdown 和图片。

# 核心概念

- 目标检测项目可按用户层、深度模型层和数据采集层理解，涉及数据采集、标注、训练、接口实现、模型部署和客户端调用。
- 目标检测算法可粗分为两步走方法和端到端方法。两步走方法先区域推荐再分类，代表包括 R-CNN、SPPNet、Fast R-CNN、Faster R-CNN；端到端方法代表包括 YOLO 和 SSD。
- 目标检测不只判断类别，还要判断目标位置，因此需要理解 bounding box、ground truth box 和 predicted box。
- R-CNN 使用 region proposal 生成候选区域，再经过 CNN 特征提取、SVM 分类、NMS 和 bbox 回归。
- SPPNet 的关键改进是整图卷积后把候选区域映射到 feature map，并通过空间金字塔池化得到固定长度特征。
- Fast R-CNN 进一步使用 RoI pooling，将分类和 bbox 回归放入更统一的流程。
- 评估目标检测时需要同时关注位置和类别，当前笔记提到 IoU、AP、mAP、准确率和召回率。
