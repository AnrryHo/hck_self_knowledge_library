# Codex

Codex 是本知识库用于本地文件处理、项目执行和知识整理的 AI 编程工具。

## 使用框架

1. **权限边界**：先确定工作目录和允许执行的操作范围。
2. **上下文规则**：用全局指令记录通用偏好，用项目级 `AGENTS.md` 记录项目约束。
3. **复杂任务**：先规划，再执行；需要隔离试验时使用 fork。
4. **能力扩展**：按任务选择插件、Skill、CLI 或 MCP。
5. **流程复用**：将稳定的多步骤流程沉淀为 Skill 或自动化任务。
6. **会话控制**：用斜杠命令和快捷键管理模型、上下文、状态、diff 与审查。
7. **云端分工**：不依赖本机文件和本地配置、需要并行或长时间后台运行的任务，可考虑交给 Codex Cloud。

## 本地与云端取舍

- **优先本地**：任务需要读取本机文件、使用本地工具、依赖本地 MCP server、涉及未提交资料或需要频繁人工确认。
- **优先云端**：任务只依赖 GitHub 仓库内文件，边界清楚、可一句话验收，且适合并行、后台长跑或直接生成 PR。
- **云端前置条件**：仓库规则需要写入 GitHub 可见文件；依赖安装、运行时版本、环境变量、Secrets 与网络白名单需要在云端环境中配置。
- **云端安全原则**：Agent 阶段默认断网；确需联网时从域名白名单与只读 HTTP 方法开始，避免提示注入导致数据外发。

## 常用会话操作

- 打 `/` 查看当前入口真实可用的斜杠命令，不要把 CLI、桌面 App、IDE 扩展的命令表混用。
- 用 `/status` 确认模型、权限、可写目录和上下文余量。
- 用 `/plan` 先规划复杂任务，用 `/compact` 压缩长会话，用 `/clear` 或 `/new` 开启新上下文。
- 提交前用 `/diff` 检查全部改动，用 `/review` 辅助审查风险。
- CLI 中 `Ctrl+L` 只清屏不丢上下文，行首 `!` 可运行 shell 命令并把输出纳入上下文。

## 在本知识库中的应用

- `AGENTS.md` 定义知识入库、复习和项目管理规则。
- `scripts/fuxi.py` 负责机械流程。
- AI 负责摘要、理解、方法提炼、关联判断和复习知识点识别。
- 新增外部资料入库时，应先保留 raw，本地生成单源笔记，再更新索引、关联和主题页。

## 注意

Codex 的模型、额度、界面、插件、命令和平台支持会更新。涉及具体产品状态时，应以 OpenAI 官方资料和当前客户端为准。
