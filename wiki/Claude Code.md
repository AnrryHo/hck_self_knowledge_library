# Claude Code

Claude Code 是 Anthropic 面向编程场景的代理式 AI 工具。根据已入库资料，它的核心定位是：读取项目上下文、调用工具、修改文件、运行命令，并在用户审查下推进真实项目任务。

## 核心定位

- **不是普通聊天框**：它不只是生成代码片段，而是能在项目目录中读取文件、修改文件和运行命令。
- **不是纯补全工具**：它更适合跨文件修改、修 bug、补测试、清理 lint、理解旧项目等需要上下文和执行循环的任务。
- **仍需人工把关**：技术取舍、业务规则、提交前 diff 审查和最终责任仍由用户承担。

## 工作模型

Claude Code 的工作方式可概括为代理循环：

1. 理解用户目标和当前项目状态。
2. 选择工具读取文件、搜索代码、修改文件或运行命令。
3. 观察工具返回结果。
4. 根据反馈继续修正，直到任务完成或需要用户介入。

用户可以在循环中打断、补充上下文、要求解释计划或纠正方向。

## 安装与环境

已入库资料强调优先使用官方原生安装脚本，而不是旧的 npm 全局安装方式。安装前需要确认系统版本、账号类型、网络条件和终端类型。Windows 用户还要区分 PowerShell 与 CMD 的命令差异。

涉及 Claude Code 最新账号要求、安装命令、版本和平台支持时，应回查官方资料；本页只记录当前入库文章中的理解。

## MCP 扩展

MCP 是 Claude Code 连接外部工具和数据源的接口。它可以把 Jira、GitHub、数据库、Figma、Sentry、Notion 等外部服务接入 Claude Code。

- 本地工具通常用 stdio server。
- 远程云服务优先用 HTTP server。
- SSE 是旧形态，新配置应避免使用。
- server 可按 local、project、user 三种作用域配置。
- 项目级 server 和首次工具调用都有批准机制。

接入第三方 server 前需要验证信任来源，抓取外部内容的 server 要警惕提示注入，敏感系统优先使用只读权限。

## 项目结构与配置

Claude Code 的配置分为项目级和用户级。项目级配置通常位于仓库中的 `./.claude/`，服务当前项目和团队协作；用户级配置位于 `~/.claude/`，服务个人偏好并跨项目生效。

- `CLAUDE.md` 和 `rules/` 更像给 Claude 阅读的项目指导。
- `settings.json` 是 Claude Code 强制执行的配置，可包含权限、hooks、默认模型等。
- `commands/`、`skills/`、`agents/` 分别对应自定义命令、技能和子代理，可按项目级或用户级放置。
- `.mcp.json` 位于项目根目录，用于团队共享 MCP server 配置。

提交边界是这类配置的核心风险点。团队共享的项目说明、共享配置、命令、规则、技能和子代理可以提交 git；`settings.local.json`、`CLAUDE.local.md`、任何密钥或 token 都不应提交。配置冲突时需要同时看优先级和字段类型：标量值通常按更具体作用域覆盖，数组值可能跨作用域合并。

## 交互操作

Claude Code 的终端界面重点看输入框、状态行和模式/权限提示。状态行中的权限模式决定它动手前是否询问用户，例如默认模式、自动接受编辑、计划模式等。

高频操作包括：

- `Esc` 中断当前回答或工具调用；输入框为空时双击 `Esc` 可进入回退菜单。
- `Ctrl+C` 用于中断或清空输入，`Ctrl+D` 用于退出会话。
- `Shift+Tab` 用于切换权限模式。
- `Ctrl+L` 用于终端显示异常时重绘屏幕。
- `@` 用于精确引用文件或目录，`!` 用于直接执行 shell 命令并把输出加入上下文。
- Enter 默认发送消息，多行输入需要使用 `\` + Enter、`Ctrl+J` 或支持环境下的 `Shift+Enter`。
