# fuxi 工作规则

本项目是用户的本地 AI 个人知识库，主要服务学习、项目管理和复习。

## 基本原则

- 所有外部资料尽量保留原始版本到 `raw/`。
- 原始资料只读，不覆盖、不改写。
- 每个外部资料原则上对应一篇 `notes/` 单源笔记。
- 机械工作交给 `scripts/fuxi.py`，语义工作由 AI 完成。
- 入库默认全自动；只有来源无法访问、内容无法解析或操作可能覆盖用户数据时才询问。
- 标签由 AI 根据内容生成，不维护固定标签表。
- 文章类资料生成 `keywords` 时控制在 3～5 个；如果原文可提炼的关键词不足 3 个，不要硬凑。
- 回答知识问题时，优先引用 `wiki/`，再读取 `notes/`，必要时回查 `raw/`。

## 用户要求入库时

1. 接收文件、文件夹、链接、图片、应用导出内容或一句话。
2. 外部资料先保存到 `raw/`；一句话灵感可直接进入 `notes/`。
3. 生成或完善 `notes/` 单源笔记。
4. 自动填写 frontmatter。
5. 更新 `INDEX.md`。
6. 运行关联更新。
7. 判断是否需要新建或更新 `wiki/` 主题页。
8. 识别值得复习的知识点，写入 `review.concepts`。

## 单源笔记要求

frontmatter 至少包含：

```yaml
---
title: 笔记标题
type: source_note
source: 原始来源
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags:
  - 标签
keywords:
  - 关键词
related: []
review:
  status: candidate
  concepts:
    - 知识点
  last_reviewed:
  next_review:
---
```

正文结构：

```markdown
# 摘要

# 核心观点

# 我的理解

# 可复用方法

# 项目关联

# 需要复习的知识点
```

不要凭空补充原文没有的信息。无法确认的判断要明确标注为推断。

## 用户要求复习某个知识点时

1. 先检索 `INDEX.md`、`wiki/`、`notes/` 和 `reviews/`。
2. 顺着相关笔记的 `related` 字段扩展必要材料。
3. 生成复习计划到 `reviews/`。
4. 计划至少包含核心概念、前置知识、复习顺序、自测题和实践任务。
5. 若库内资料不足，明确指出缺口，不用通用知识伪装成库内结论。

## 用户询问需要复习什么时

1. 运行 `scripts/fuxi.py review-list`。
2. 结合当前项目相关性、知识点出现频率、上次复习时间和是否缺少实践进行排序。
3. 返回精简列表，每项说明推荐复习的原因。
4. 用户要求保存时，写入 `reviews/YYYY-MM-DD-review-list.md`。

## 用户询问项目事项时

- 检索 `type: task_note`、项目标签和相关主题页。
- 明确区分资料中的事实、用户任务和 AI 建议。
- 不擅自修改完成状态、截止日期或优先级。
