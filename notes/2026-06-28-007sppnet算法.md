# 摘要

这篇为知笔记介绍 SPPNet 算法，原文标题写作“sspNet”。原文说明 SPPNet 是对 R-CNN 的改进：R-CNN 的两千个候选框都要经过卷积，所以很慢；SPPNet 先对整张图做一次卷积得到 feature map，再把候选框区域映射到特征图区域，得到特征后用于分类。截图中还展示了 SPPNet 相比 R-CNN 的差异、候选区域映射方法，以及 spatial pyramid pooling。

# 核心观点

1. R-CNN 对每个候选区域分别做 CNN，计算开销大。
2. SPPNet 先让整张图片经过 CNN 得到整图 feature map。
3. 候选区域通过映射关系对应到 feature map 上的区域。
4. 截图中提到 spatial pyramid pooling 会把候选区域特征图转换为固定大小的特征向量。
5. 原文指出 SPPNet 仍有训练慢、效率低、特征需要写入磁盘、分阶段训练和反向传播效率低等缺点。

# 我的理解

SPPNet 的关键改进是减少重复卷积：从“每个候选框都卷积一次”改为“整图卷积一次，再映射候选框”。但原文也强调，它没有彻底解决训练流程割裂和效率问题，因此是从 R-CNN 到 Fast R-CNN 之间的中间改进。

# 可复用方法

- 复习 SPPNet 时先对比 R-CNN 的重复卷积问题。
- 按“整图 CNN、候选区域映射、SPP 层、固定长度特征向量、分类”的顺序描述流程。
- 记住 SPPNet 的优点是减少卷积重复，缺点是训练仍分阶段且效率不高。
