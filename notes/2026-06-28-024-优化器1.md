# 摘要

这篇为知笔记介绍 PyTorch 优化器。原文指出优化器位于 `torch.optim` 模块下，示例是在上一节代码基础上加上优化器代码；还说明模型要对数据进行多次学习，因此需要循环，让模型对同一个数据进行多次学习。

# 核心观点

1. PyTorch 优化器来自 `torch.optim` 模块。
2. 优化器代码通常接在损失函数和反向传播之后。
3. 模型训练需要循环多次学习数据。
4. 原文示例通过循环让模型重复学习同一数据。

# 我的理解

优化器负责根据梯度更新模型参数。它和 loss/backward 共同构成训练闭环：前向计算、计算损失、反向传播、优化器更新。

# 可复用方法

- 从 `torch.optim` 中选择优化器。
- 将优化器加入已有训练代码。
- 用循环控制模型对数据的多次学习。
- 复习时把优化器放在 loss 和 backward 后面理解。

# 项目关联

这篇笔记连接损失函数与完整训练套路，是 PyTorch 训练流程中的参数更新环节。
