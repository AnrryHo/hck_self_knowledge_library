# 摘要

这是一份以中医临床诊疗术语证候问答为案例的 RAG 项目课件。它使用 LlamaIndex、本地 Hugging Face 嵌入模型和大语言模型，覆盖环境配置、模型下载、语料准备、索引构建与持久化、检索追踪、底层查询流程和自定义 Prompt。

# 核心观点

1. 项目通过外部中医知识语料补充大模型知识，展示 RAG 从建库到问答的基本工程流程。
2. 本地模型方案使用 Hugging Face Embedding 与 LLM，强调模型权重下载和运行环境配置。
3. 向量索引需要持久化，否则重复运行时会反复计算嵌入。
4. 调试 RAG 时应追踪实际命中的文档片段，并检查格式化 Prompt、Retriever 和 Response Synthesizer。
5. 通过自定义 Prompt 可以约束回答方式，但不能替代可靠数据、检索质量和领域验证。

# 我的理解

该项目的主要教学价值是把抽象的 RAG 组件映射到一个完整案例，并展示如何下钻到检索器和响应合成器。课件内容不应被视为真实临床诊疗依据。

# 可复用方法

- 在建库前明确语料来源、切分方式和元数据。
- 将索引持久化，并验证重新加载后的检索结果一致性。
- 输出命中文档及相似度，检查答案是否有实际证据支撑。
- 分别调试 Query、Prompt、Retriever、Postprocessor 和 Synthesizer。
- 医疗领域应用必须增加来源审查、专家验证、风险提示和使用边界。

# 项目关联

这是项目型学习资料，可用于理解如何把 LlamaIndex、Embedding、LLM 和本地语料组合成可运行的 RAG 原型。

# 需要复习的知识点

- RAG 建库、检索和生成的完整流程。
- 本地模型部署相较 API 方案增加了哪些工作。
- LlamaIndex 索引如何持久化和重新加载。
- 如何追踪文档片段并定位检索错误。
- 医疗 AI 原型与临床工具之间的边界。
