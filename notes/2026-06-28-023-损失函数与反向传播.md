# 摘要

这篇为知笔记介绍损失函数与反向传播。原文记录 `L1Loss` 是线性损失函数，可通过 `reduction` 参数指定损失计算方式；还提到交叉熵 `CrossEntropyLoss`。原文总结损失函数有两种作用：计算实际输出和目标之间的差距，为更新输出提供依据，也就是反向传播。示例中使用 `result_loss.backward()`。

# 核心观点

1. `L1Loss` 可用于线性损失计算。
2. `reduction` 参数控制损失函数的计算方式。
3. `CrossEntropyLoss` 是另一种损失函数。
4. 损失函数既衡量输出与目标的差距，也为反向传播提供依据。
5. `backward()` 用于触发反向传播。

# 我的理解

损失函数是模型训练的反馈信号。只有计算出 loss 并进行 `backward()`，优化器才有梯度依据更新参数。

# 可复用方法

- 根据任务选择合适损失函数。
- 设置损失函数时关注 `reduction`。
- 前向输出后先计算 loss。
- 调用 `result_loss.backward()` 进行反向传播。
