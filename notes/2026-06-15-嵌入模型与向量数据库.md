# 摘要

这份 Jupyter 课件系统介绍向量表征、文本嵌入、相似度计算、嵌入模型和向量数据库，并通过 OpenAI Embedding 与 Chroma 展示向量生成、存储和检索流程，最后给出 Milvus 扩展学习方向。

# 核心观点

1. 向量表征将文本等非结构化对象映射为可计算的数值向量，使语义相似度计算成为可能。
2. 文本嵌入模型负责生成向量，模型选择需要考虑语种、效果、成本、维度和部署方式。
3. 向量检索依赖距离或相似度度量，课件涉及余弦相似度的计算。
4. 向量数据库围绕嵌入向量提供存储、索引和相似性检索能力，与传统数据库的查询方式不同。
5. Chroma 可通过客户端、集合、嵌入函数和查询接口构成基础检索流程，并支持 Client-Server 模式。

# 我的理解

Embedding 与向量数据库分别解决“如何表示语义”和“如何高效保存、查找语义表示”两个问题，是 RAG 检索层的基础设施。该归纳来自课件的章节顺序与示例代码。

# 可复用方法

- 先根据数据语言、领域和部署约束选择嵌入模型。
- 统一文档与查询所用的嵌入模型和向量维度。
- 用明确的相似度指标验证近邻结果，不只检查接口是否返回数据。
- 先用 Chroma 验证小规模流程，再根据数据量和部署需求评估 Milvus 等方案。
- 将写入、持久化、加载和查询分别测试，避免把数据问题与检索问题混在一起。
