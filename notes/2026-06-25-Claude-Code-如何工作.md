# 摘要

这篇文章解释 Claude Code 为什么能像代理一样完成任务。作者把它的工作方式概括为“想、做、看”的循环：先理解目标和当前项目状态，再调用工具执行动作，随后观察结果并决定下一步。

文章强调用户也在循环中：可以随时打断、纠正方向、补充上下文或要求它先解释计划。Claude Code 能真正动手，是因为它具备读取文件、编辑文件、运行命令、搜索代码、查看 Git 状态等工具能力。

# 核心观点

- Claude Code 的工作不是一次性回答，而是在代理循环里持续推进：分析目标、选择工具、执行动作、观察反馈、继续修正。
- 用户不是旁观者，而是随时可以介入的协作者；当方向不对、需求不清或风险变高时，应及时打断。
- 工具调用是它区别于普通聊天模型的关键。模型本身负责判断和规划，工具负责接触真实项目环境。
- 它能看到的内容来自当前工作目录、用户提供的信息、命令输出和工具返回结果；不是天然知道电脑上的所有内容。
- 让它处理任务时，最好从小而清晰的目标开始，并在执行前要求它说明将要检查哪些文件、准备怎么做。

# 我的理解

这篇文章适合用来理解“AI Agent”在编程场景中的基本运行模型。所谓代理能力并不是模型突然拥有自主意识，而是模型在权限边界内反复选择工具、读取反馈、修正行动。

对个人知识库项目也有启发：复杂入库或复习任务可以被看成类似循环，先检索资料，再生成笔记，再跑脚本验证，再根据结果修正。区别在于本项目规则要求原始资料只读、语义判断由 AI 完成、机械更新交给 `scripts/fuxi.py`。

# 可复用方法

- 给代理式工具下任务时说明目标、边界、验收标准和允许操作范围。
- 在高风险任务前要求它先列计划，再执行。
- 利用中途打断机制纠偏，而不是等它跑完整个错误方向。
- 根据工具返回结果做下一步决策，不把模型的初始判断当作最终结论。
