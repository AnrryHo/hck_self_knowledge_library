# 摘要

这篇为知笔记介绍完整 R-CNN 架构。原文正文写到“完整的rcnn架构”，截图展示从 image 到 region proposal、warped region、CNN、SVM 分类和 bbox regressor 的流程。截图中的步骤还可辨认出选择性搜索产生约 2000 个候选区域、缩放到 AlexNet 输入大小、用 CNN 提取特征、用 SVM 分类、用非极大值抑制筛除重叠建议框，以及 bbox 回归微调检测框。

# 核心观点

1. R-CNN 使用区域推荐方法生成候选区域，而不是直接遍历所有窗口。
2. 候选区域会被变换为固定大小后输入 CNN 提取特征。
3. 每个候选区域的 CNN 特征会送入 SVM 分类器判断类别。
4. R-CNN 还使用 NMS 处理重叠候选框，并使用 bbox 回归微调位置。

# 我的理解

R-CNN 的核心是把目标检测拆成多个步骤：先找候选框，再提特征，再分类，再做框回归。基于截图可推断，课程强调 R-CNN 是后续 SPPNet、Fast R-CNN、Faster R-CNN 的基础检测思路。

# 可复用方法

- 复述 R-CNN 时按“候选区域、固定尺寸变换、CNN 特征、SVM 分类、NMS、bbox 回归”的顺序组织。
- 将 region proposal 与后续分类器分开理解。
- 学习后续改进算法时，重点比较它们减少了 R-CNN 哪些重复计算或分阶段训练成本。
