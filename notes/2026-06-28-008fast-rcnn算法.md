# 摘要

这篇为知笔记介绍 Fast R-CNN 算法。原文指出 Fast R-CNN 是对 R-CNN 的第二种改进，其中 RoI pooling 是关键。截图展示流程：整张图片输入基础卷积网络得到 feature map，region proposal 映射到 feature map，经 RoI pooling layer 形成 RoI feature vector，再进入全连接层，分别连接 softmax 分类和 bbox regressor。

# 核心观点

1. Fast R-CNN 先对整张图片做卷积，得到整图 feature map。
2. 候选区域会映射到 feature map 上，而不是每个候选框单独卷积。
3. RoI pooling 把不同大小的候选区域特征变成固定长度特征向量。
4. 截图中 softmax 用于分类，bbox regressor 用于边框回归。
5. 原文强调 RoI pooling 是 Fast R-CNN 的关键。

# 我的理解

Fast R-CNN 进一步整合了 R-CNN 系列的流程。基于截图可推断，它沿用了候选区域思想，但把特征提取、RoI pooling、分类和 bbox 回归放入更统一的模型中，从而减少 R-CNN 和 SPPNet 中的重复计算与分阶段处理。

# 可复用方法

- 描述 Fast R-CNN 时按“整图卷积、RoI 映射、RoI pooling、全连接、softmax 分类、bbox 回归”的顺序复述。
- 对比 R-CNN 和 SPPNet，重点看重复卷积和分阶段流程是否减少。
- 把 RoI pooling 当作理解 Fast R-CNN 的核心概念。
