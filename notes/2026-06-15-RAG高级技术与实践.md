# 摘要

这份课件从 RAG 技术树出发，讲解 RAFT、Top-K、索引改进、重排序、查询扩展和索引扩展等召回优化方法，并演示使用 Qwen-Agent 构建 RAG、使用 Ragas 评估质量，以及商业项目中的工程实施步骤。

# 核心观点

1. RAG 的质量不是只由大模型决定，索引、查询、召回、排序和生成各环节都会影响最终答案。
2. Top-K 需要结合语料和问题调整，过少可能漏召回，过多会引入无关上下文。
3. 重排序、Multi-Query、父文档检索和查询改写可以改善基础向量召回的不足。
4. RAG 评估需要同时观察问题、上下文和答案之间的关系，而不是只做人工主观判断。
5. 课件通过 Qwen-Agent、LangChain、FAISS/Chroma 和 Ragas 展示从构建到评估的完整链路。

# 我的理解

高级 RAG 的重点是建立可测量的检索优化循环：先明确失败发生在召回、排序还是生成，再选择对应策略并用评估数据验证。该判断是对课件技术树和评估章节的归纳。

# 可复用方法

- 建立一组代表性问题及参考答案，作为优化前后的固定测试集。
- 分别记录召回文档、排序结果和最终答案，以定位错误环节。
- 先调节 Top-K 和切分策略，再逐步加入查询改写、父文档检索和重排序。
- 使用 Ragas 等评估框架形成自动指标，同时保留人工抽检。
- 商业落地时把数据治理、评测、监控和迭代纳入工程流程。
