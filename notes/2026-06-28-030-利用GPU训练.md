# 摘要

这篇为知笔记介绍 PyTorch 利用 GPU 训练。原文提到可以加 `cuda()` 方法，并应使用 `torch.cuda.is_available` 判断是否存在 GPU；也可以在 Google Colab 中使用 GPU；可用 `nvidia-smi` 查看配置，如果在 Python 环境中运行命令，需要在命令前加感叹号；引入 `time` 包可以得到训练时间。原文还指出更常用的 GPU 使用方式是 `.to`。

# 核心观点

1. 使用 GPU 前需要判断 CUDA 是否可用。
2. `cuda()` 可以把相关对象放到 GPU 上。
3. `.to` 是原文记录的更常用 GPU 使用方式。
4. Colab 可用于 GPU 训练。
5. `nvidia-smi` 可查看 GPU 配置，`time` 可统计训练耗时。

# 我的理解

GPU 训练不是只移动模型，还需要确保模型、输入数据、标签和损失相关张量在同一设备上。原文提到“上面 4 种情况加上 to 的代码”，具体四种情况需要回查 raw 截图确认。

# 可复用方法

- 先用 `torch.cuda.is_available()` 判断 GPU。
- 统一用 `.to(device)` 管理模型和数据设备。
- 在 Colab 或本机训练前用 `nvidia-smi` 查看 GPU。
- 用 `time` 记录训练耗时，比较 CPU/GPU 效果。
