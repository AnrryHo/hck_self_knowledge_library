# 摘要

这是一篇面向初学者的 Codex 桌面端使用教程，覆盖安装登录、上下文与额度、模型选择、本地文件权限、命令行、持久记忆、复杂项目规划、插件、Skills、CLI、MCP 和自动化任务。

# 核心观点

1. Codex 可以围绕本地项目执行文件读写、命令行操作和多任务协作，实际能力受项目权限模式约束。
2. 文档把权限分为默认权限、自动审查和完全访问权限，并强调权限越高，Codex 可自主完成的操作越多。
3. 持久指令可按作用域拆分：全局自定义指令适合通用偏好，项目级 `AGENTS.md` 适合特定项目规则。
4. 面对复杂或需求不清晰的项目，可以先使用计划模式澄清目标，再执行、批注、调整或通过 fork 测试支线方案。
5. Codex 的能力可通过插件、Skills、CLI 和 MCP 扩展，并可把重复流程沉淀为自动化任务。
6. 文档建议将成熟的多步骤工作流封装为 Skill，以便重复调用。

时效性提示：文中有关具体模型名称、速度倍率、额度、插件清单、平台限制和界面位置的描述属于来源在导出时的说法，可能随产品更新而变化，使用前应查阅官方资料。

# 我的理解

这篇资料的重点不是某个单独功能，而是一套逐层增强的 Codex 使用路径：

1. 先掌握项目目录、权限和上下文。
2. 再用 `AGENTS.md` 固化协作规则。
3. 用计划模式管理复杂任务。
4. 用插件、Skills、CLI 和 MCP 补充工具能力。
5. 最后把稳定流程变成 Skill 或自动化任务。

上述分层是根据原文章节组织做出的归纳。

# 可复用方法

- 开始项目前先选择工作目录并确认权限边界。
- 将跨项目通用偏好放入全局指令，将项目约束写入项目根目录的 `AGENTS.md`。
- 对复杂任务先要求生成计划，确认后再执行。
- 对不确定方案使用 fork 建立副本验证，避免干扰主任务。
- 对重复出现的“检索、筛选、写作、润色、发布”流程，整理为 Skill。
- 需要连接外部系统时，根据场景选择插件、CLI 或 MCP。
- 设置自动化任务时明确执行时间、调用流程、输出格式和投递位置。

# 项目关联

本知识库已经使用项目级 `AGENTS.md` 约束入库、复习和项目事项处理流程，与文档中的“项目级持久记忆”方法直接相关。

当前 `scripts/fuxi.py` 承担机械入库、索引、关联和复习列表等工作，也符合“稳定流程工具化”的思路。

- Skills、CLI、插件和 MCP 各自解决什么问题。
- 计划模式、执行阶段批注和 fork 的适用时机。
- 如何把一次性操作整理成可重复运行的 Skill 或自动化任务。
