# fuxi 技术栈初稿

## 1. 目标约束

这套技术栈只服务一个目标：把你的个人知识库做成一个可长期使用的、本地优先的、全自动入库和检索系统。

已确认的约束：

- 输入来源包括文件夹、文件夹下的文档、指定应用的文档或收藏夹、图片、链接、一句话输入。
- 知识库主要服务学习、项目管理、复习。
- `raw` 原文需要保留。
- 标签由 AI 自动判断，不维护固定标签表。
- 入库默认全自动，不要求人工逐步确认。

## 2. 总体原则

- 主流优先。
- 本地优先。
- 文件优先。
- 脚本优先。
- AI 负责语义，脚本负责机械活。
- 能少一个组件就少一个组件。

## 3. 推荐主栈

### 3.1 语言

- `Python`

理由：

- AI 最熟。
- 生态足够大。
- 适合写抓取、解析、索引、关联、复习计划这类脚本。

### 3.2 存储

- `Markdown` 文件作为主存储。
- `YAML frontmatter` 作为元数据。
- `raw/`、`notes/`、`wiki/`、`briefs/`、`reviews/` 作为目录分层。

理由：

- 人能直接读。
- Git 能 diff。
- AI 能直接处理。
- 不需要数据库就先不用。

### 3.3 搜索

- `ripgrep (rg)`
- 文件名、标题、标签、关键词检索

理由：

- 本地快。
- 简单可解释。
- 对个人知识库已经足够。

### 3.4 入库编排

- Python 入库脚本
- 本地定时任务或目录监听
- 目录扫描器

建议组合：

- `watchdog` 用来监听本地目录变化。
- 系统任务计划或定时扫描做兜底。

理由：

- 你希望全自动，所以需要一个稳定的本地触发机制。
- 目录监听适合“文件一落地就处理”。
- 定时扫描适合补漏。

### 3.5 原文抽取

- 网页正文抽取库
- 本地文件解析器
- PDF 转文本工具
- 图片 OCR / 视觉识别

建议方向：

- 网页：使用成熟的正文抽取方案。
- PDF：先转文本，再交给 AI。
- 图片：先 OCR，必要时再让 AI 复核。

### 3.6 AI 层

- Claude 或 Codex 这类命令行可调用模型
- 用于摘要、打标签、关联判断、主题编译、复习计划生成

理由：

- 你的需求本质是语义处理。
- AI 比脚本更适合做归纳、判断和生成。

### 3.7 复习层

- `reviews/` 目录
- 复习状态字段
- `last_reviewed`
- `next_review`
- `review.status`

用途：

- 生成单个知识点的复习计划。
- 生成待复习知识点列表。
- 记录复习结果和下次复习时间。

## 4. 分层选型

### 4.1 进

输入类型与处理方式建议如下：

| 输入 | 处理方式 |
|---|---|
| 文件夹 | 扫描目录，逐个文件入库 |
| 文件夹下文档 | 逐个抽取内容，生成单源笔记 |
| 指定应用文档或收藏夹 | 优先通过导出文件或 API 获取原始内容 |
| 图片 | OCR + AI 语义提炼 |
| 链接 | 拉取正文，保存 raw，再生成笔记 |
| 一句话 | 直接生成想法笔记 |

### 4.2 存

推荐最小存储结构：

```text
raw/
notes/
wiki/
briefs/
reviews/
INDEX.md
```

说明：

- `raw/` 放原始内容。
- `notes/` 放单源笔记。
- `wiki/` 放主题编译页。
- `briefs/` 放选题、行动建议、周报。
- `reviews/` 放复习计划和待复习列表。

### 4.3 取

取数优先级建议：

1. `INDEX.md`
2. `rg` 关键词检索
3. `tags` / `keywords`
4. `related`
5. `wiki/` 主题页
6. AI 综合回答

## 5. 自动化流程建议

### 5.1 入库流水线

```text
输入源
  ↓
原文抽取
  ↓
保存 raw
  ↓
AI 生成 notes
  ↓
AI 打标签 / 关键词 / 摘要
  ↓
更新 INDEX
  ↓
计算 related
  ↓
更新 wiki
  ↓
识别复习知识点
  ↓
更新 reviews
```

### 5.2 复习流水线

#### 指定知识点复习

```text
用户指定知识点
  ↓
AI 检索相关 notes / wiki / reviews
  ↓
生成复习计划
  ↓
保存到 reviews/
```

#### 待复习列表

```text
用户询问最近该复习什么
  ↓
AI 读取最近学习、长期未复习、项目相关的内容
  ↓
输出复习列表
  ↓
必要时写入 reviews/
```

## 6. 关系判断策略

MVP 阶段建议继续用轻量规则，不上向量库。

规则优先级：

- 标签重叠。
- 标题关键词重叠。
- AI 复核。

这样做的原因：

- 可解释。
- 维护简单。
- 对个人知识库已经足够。

## 7. 暂不建议上来的组件

- 数据库。
- 向量库。
- 复杂 RAG 框架。
- Web 前端。
- 多端同步。
- 消息队列。

这些东西不是不能用，而是对你当前这个阶段的边际收益不高，先不加。

## 8. 你后面最可能要决定的点

- 图片识别到底是只做 OCR，还是让 AI 直接读图。
- 指定应用具体有哪些，以及它们能否导出统一格式。
- 文件夹输入是监听目录，还是定时扫描目录。
- `reviews/` 里是否要记录每次复习完成情况。
- `wiki/` 主题页是手动更新，还是自动触发更新。
